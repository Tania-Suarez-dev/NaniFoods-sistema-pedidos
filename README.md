NaniFoods - Sistema de Gestión de Pedidos
Este proyecto es una aplicación web desarrollada para gestionar pedidos en una cocina tipo *dark kitchen*. La idea fue centralizar tanto la gestión de productos como el control de ventas y pedidos en un solo sistema.

Funcionalidades:

- Gestión de pedidos de productos  
- Sistema de reseñas de usuarios  
- Panel administrativo para control del sistema  
- Dashboard con métricas como:
  - Productos más vendidos  
  - Ingresos generados  
- Visualización de datos mediante gráficas  
- CRUD de productos  
- Generación de reportes en Excel


Tecnologías utilizadas:

- PHP  
- MySQL (si usaste base de datos)  
- HTML  
- CSS  
- Bootstrap  
- JavaScript  

Objetivo del proyecto:
El proyecto busca facilitar la gestión de pedidos y permitir tener una visión más clara del comportamiento de las ventas y los productos.


Lo que trabajé en este proyecto:

- Desarrollo de lógica para gestión de pedidos  
- Manejo de datos y conexión con base de datos  
- Construcción de CRUDs  
- Generación de reportes  
- Estructuración de interfaz para usuarios y administrador  


Desarrollo:

Este proyecto fue desarrollado en equipo como parte del proceso de formación.
Mi enfoque principal estuvo en la implementación de la lógica del sistema, manejo de datos y construcción de funcionalidades como el panel administrativo y la gestión de pedidos.
